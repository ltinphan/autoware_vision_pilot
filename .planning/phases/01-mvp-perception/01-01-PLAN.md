<threat_model>
## Trust Boundaries

| Boundary | Description |
|---|---|
| camera files (Media/) -> CLI | Untrusted input image files may be crafted to cause memory or format issues |
| model checkpoints -> CLI | Checkpoints are untrusted binaries; loading arbitrary torch checkpoints can execute code during deserialization |
| CLI -> filesystem | Outputs written to disk may overwrite or expose sensitive files if out_dir path not validated |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
| T-PER-01 | Spoofing | Input images | mitigate | Validate image headers with PIL, reject extremely large images (>16MP) and enforce expected resolution where applicable (per-model checks) |
| T-PER-02 | Tampering | Model checkpoints | mitigate | Do not auto-execute checkpoint code; require user-provided checkpoints and document checksums. Where possible, load weights via torch.load(map_location=...) and avoid pickle-based object execution (use weights_only loading patterns). |
| T-PER-03 | Repudiation | Logs | mitigate | CLI should emit structured logs (timestamped) and write a brief run_manifest.json with command line, model id, checkpoint path (not contents), input files processed, and exit status. |
| T-PER-04 | Information disclosure | Outputs | mitigate | Validate and constrain out_dir paths to within project sandbox unless absolute path explicitly confirmed; warn when writing outside repo. |
| T-PER-05 | Denial of Service | Resource exhaustion | mitigate | Limit concurrent image processing (single-threaded by default), reject inputs above size threshold, enforce timeouts for model loading/inference where possible. |
| T-PER-06 | Elevation of privilege | File permissions | accept | CLI runs with user privileges; document that root use is unsupported and warn in ONBOARDING.md. |

## Safety Checklist (Perception-specific)
- [ ] Validate input sizes and types before passing to model (PIL verification)
- [ ] Fail closed: if model output is malformed, do not write partially-formed outputs — write to temp file then atomically move
- [ ] Log model/device used and inference time for traceability
- [ ] Avoid deserializing arbitrary objects from checkpoints; prefer weight-only loading where the model architecture is imported from code in repo
- [ ] Document that model checkpoints are untrusted and provide checksum guidance in ONBOARDING.md
</threat_model>
